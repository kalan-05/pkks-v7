# Требования к хостингу

Сайту нужен PHP-хостинг. Если хостинг не поддерживает PHP, JSON-render и админ-панель работать не будут.

## Требуется сейчас

| Требование | Зачем нужно |
| --- | --- |
| PHP 8.2 или выше | Для запуска `index.php` и файлов админ-панели. |
| Включённое расширение JSON | Для чтения и записи `data/team.json`, `data/services.json` и `data/prices.json`. |
| PHP sessions | Для авторизации в `/admin/`. |
| Поддержка `password_hash` и `password_verify` | Для проверки пароля администратора по hash. |
| Поддержка `random_bytes` и `hash_equals` | Для CSRF-токенов и безопасного сравнения значений. |
| Возможность выполнять `index.php` | Главная страница должна открываться через PHP-entrypoint. |
| Возможность читать `data/*.json` из PHP | Публичная часть должна получать данные сотрудников, услуг и цен. |
| Возможность писать в разрешённые JSON и runtime-папки | Админка должна сохранять данные, создавать backup, audit log и rate-limit файл. |
| Поддержка UTF-8 | Чтобы русская кириллица не превращалась в битый текст. |
| Возможность задать приоритет `index.php` перед `index.html` | Корень сайта `/` должен открывать PHP-render, а не fallback-HTML. |
| Возможность закрыть прямой web-доступ к служебным папкам | `config/`, `storage/`, `data/backups/` и `includes/` не должны скачиваться из браузера. |

## Writable-пути

На хостинге PHP-процессу нужны минимальные права, достаточные для записи в:

- `data/team.json`
- `data/services.json`
- `data/prices.json`
- `data/backups/team/`
- `data/backups/services/`
- `data/backups/prices/`
- `storage/logs/`
- `storage/login-attempts.json`

Не используйте `777` как стандартную настройку. Права должны быть минимальными и достаточными именно для пользователя, под которым выполняется PHP на конкретном хостинге.

## Admin config

Для включения входа в админку нужен файл:

```text
config/admin-auth.php
```

Его создают на хостинге вручную из:

```text
config/admin-auth.php.example
```

В `config/admin-auth.php` должны быть:

- логин администратора;
- `admin_password_hash`, сгенерированный через PHP;
- параметры таймаутов сессии;
- параметры rate-limit входа.

Файл `config/admin-auth.php` не должен попадать в репозиторий, публичные архивы или открытые URL.

## Apache-compatible hosting

Если хостинг использует Apache и разрешает `.htaccess`, текущие файлы помогают настроить базовую защиту:

- корневой `.htaccess` задаёт `DirectoryIndex index.php index.html`;
- `admin/.htaccess` задаёт `DirectoryIndex index.php login.php`;
- `includes/.htaccess` закрывает прямой доступ к `includes/`;
- `storage/.htaccess` закрывает прямой доступ к `storage/`.

Отдельно нужно проверить запрет прямого доступа к:

- `config/admin-auth.php`;
- `data/backups/`.

Если этих запретов нет, их нужно настроить в панели хостинга или через поддержку хостинга.

## Non-Apache hosting

Если хостинг использует Nginx или другой сервер без `.htaccess`, файлы `.htaccess` не применяются.

Тогда нужно отдельно настроить:

- приоритет `index.php` перед `index.html`;
- запрет прямого доступа к `config/admin-auth.php`;
- запрет прямого доступа к `storage/`;
- запрет прямого доступа к `data/backups/`;
- запрет прямого доступа к `includes/`.

Конкретный `server block` или правила панели зависят от хостинга. Их должен настроить владелец хостинга или техническая поддержка хостинга.

## Желательно, но не критично

- Доступ к PHP error log.
- Запрет directory listing на уровне хостинга.
- Удобная настройка порядка index-файлов в панели хостинга.
- Возможность быстро менять права на файлы и папки.
- Возможность сделать резервную копию файлов перед ручным rollback.

## Что уточнить у хостинга перед загрузкой

- Какая версия PHP доступна для сайта.
- Включено ли расширение JSON.
- Работают ли PHP sessions.
- Поддерживаются ли `password_hash`, `password_verify`, `random_bytes`.
- Как в панели задаётся приоритет `index.php` перед `index.html`.
- Поддерживается ли `.htaccess`.
- Как закрыть прямой web-доступ к `config/`, `storage/`, `data/backups/` и `includes/`, если `.htaccess` не поддерживается.
- Под каким пользователем PHP пишет файлы и какие минимальные права нужны для writable-путей.
- Где смотреть PHP errors, если публичная часть или админка вернёт 500.
