# Требования к хостингу

Сайту нужен PHP-хостинг. Если хостинг не поддерживает PHP, JSON-рендер и будущий личный кабинет работать не будут.

## Требуется уже сейчас

| Требование | Зачем нужно |
| --- | --- |
| PHP 8.2 или выше | Для запуска `index.php` и PHP-render текущих блоков сайта. |
| Включённое расширение JSON | Для чтения `data/team.json`, `data/services.json` и `data/prices.json`. |
| Возможность выполнять `index.php` | Главная страница должна открываться через PHP-entrypoint. |
| Возможность читать `data/*.json` из PHP | PHP-код должен получать данные сотрудников, услуг и цен. |
| Поддержка UTF-8 | Чтобы русская кириллица не превращалась в битый текст. |
| Возможность задать приоритет `index.php` перед `index.html` | Корень сайта `/` должен открывать PHP-render, а не fallback-HTML. |
| Возможность закрыть прямой доступ к `includes/` | Вспомогательные PHP-файлы не должны открываться из браузера напрямую. |

## Потребуется позже для админки

Будущий личный кабинет потребует дополнительных возможностей хостинга:

- PHP sessions.
- Безопасная авторизация.
- Права записи в `data/*.json`.
- Права записи в `data/backups/`.
- Возможность создавать backup перед сохранением данных.
- Возможность вести audit log.
- Защита `admin/`, `data/backups/` и `logs/` от публичного доступа.

Эти возможности сейчас не используются, потому что админка ещё не создана.

## Желательно, но не критично

- Apache-compatible `.htaccess`.
- Запрет directory listing на уровне хостинга.
- Удобная настройка порядка index-файлов в панели хостинга.
- Доступ к логам ошибок PHP.

Если хостинг использует Apache и разрешает `.htaccess`, текущий файл `.htaccess` уже задаёт:

```apache
DirectoryIndex index.php index.html
```

Если хостинг использует Nginx, `.htaccess` не применяется. Для Nginx нужна настройка принципа:

```nginx
index index.php index.html;
```

Конкретную настройку Nginx должен выполнить владелец хостинга или техническая поддержка хостинга.

## Что уточнить у хостинга перед загрузкой

- Какая версия PHP доступна для сайта.
- Включено ли расширение JSON.
- Как в панели задаётся приоритет `index.php` перед `index.html`.
- Поддерживается ли `.htaccess`.
- Как закрыть прямой доступ к папке `includes/`, если `.htaccess` не поддерживается.
- Где смотреть PHP errors, если `index.php` вернёт 500.
