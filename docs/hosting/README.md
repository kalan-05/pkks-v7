# Handoff-пакет для установки сайта на хостинг

Этот пакет описывает, как передать сайт `pkks-v7` / «Правовая контора К. Сопрачева» на PHP-хостинг заказчика.

Это не deploy-инструкция под конкретный хостинг. Хостинг заказчика заранее неизвестен, доступа к production нет, загрузка на сервер в этом контуре не выполняется.

Это универсальный handoff-pack для установки на PHP-хостинг.

## Для кого этот пакет

- Для заказчика, который будет передавать файлы сайта своему хостингу или подрядчику.
- Для технической поддержки хостинга, если потребуется настроить приоритет `index.php`.
- Для следующего этапа разработки, где будет проектироваться личный кабинет.

## Текущая архитектура сайта

Сайт сейчас состоит из PHP-entrypoint, fallback-HTML и статических ассетов:

- `index.php` читает JSON-файлы из `data/` и рендерит динамические блоки.
- `data/team.json` содержит сотрудников.
- `data/services.json` содержит услуги.
- `data/prices.json` содержит цены.
- `includes/*.php` содержит вспомогательный PHP-код для чтения JSON и рендера блоков.
- `index.html` оставлен как fallback/reference-версия сайта.
- `.htaccess` задаёт приоритет `DirectoryIndex index.php index.html` для Apache-compatible hosting.
- `includes/.htaccess` закрывает прямой доступ к PHP-файлам в `includes/` на Apache-compatible hosting.
- `css/`, `js/`, `img/`, `font/` содержат стили, скрипты, изображения и шрифты.

## Что уже готово

- PHP-render для блоков «О нас», «Услуги» и «Цена».
- JSON-источники данных для сотрудников, услуг и цен.
- Fallback через `index.html`.
- Корневой `.htaccess` для приоритета `index.php` перед `index.html`.
- Защита `includes/` от прямого доступа на Apache-compatible hosting.

## Что не входит в этот пакет

- Загрузка на production.
- Настройка DNS.
- Подключение к конкретной панели хостинга.
- Изменение PHP-кода, JSON, дизайна, SEO или маршрутов.
- Создание админки, авторизации, форм и записи в JSON.
- Настройка конкретного Nginx `server block`.

## Какие файлы сайта нужны на хостинге

В корне сайта должны оказаться:

- `index.php`
- `index.html`
- `.htaccess`
- `css/`
- `js/`
- `img/`
- `font/`
- `data/`
- `includes/`

## Краткий порядок установки

1. Проверить, что хостинг поддерживает PHP 8.2 или выше и JSON.
2. Загрузить файлы сайта в корень сайта на хостинге.
3. Убедиться, что `/` открывает PHP-версию через `index.php`.
4. Проверить `/index.php` и `/index.html` напрямую.
5. Проверить, что сотрудники, услуги и цены отображаются из JSON.
6. Проверить, что `includes/*.php` не открываются напрямую.
7. Если `/` открывает старый `index.html`, настроить приоритет `index.php` перед `index.html` через хостинг-панель или поддержку хостинга.
