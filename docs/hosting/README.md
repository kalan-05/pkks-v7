# Handoff-пакет для установки сайта и админ-панели на хостинг

Этот пакет описывает, как передать сайт `pkks-v7` / «Правовая контора К. Сопрачева» на PHP-хостинг заказчика.

Это не deploy-инструкция под конкретный хостинг. Хостинг заказчика заранее неизвестен, доступа к production нет, загрузка на сервер в этом контуре не выполняется.

Пакет фиксирует текущее состояние проекта: публичный PHP-render и рабочую MVP-админку для редактирования сотрудников, фото сотрудников, услуг и цен.

## Для кого этот пакет

- Для заказчика, который будет передавать файлы сайта своему хостингу или подрядчику.
- Для технической поддержки хостинга, если потребуется настроить PHP, права записи и запрет доступа к служебным файлам.
- Для следующего этапа разработки, где админка будет расширяться за пределы MVP.

## Текущая архитектура сайта

Сайт сейчас состоит из PHP-entrypoint, JSON-данных, MVP-админки и статических ассетов:

- `index.php` читает JSON-файлы из `data/` и рендерит динамические блоки.
- `data/team.json` содержит сотрудников.
- `data/services.json` содержит услуги.
- `data/prices.json` содержит цены.
- `admin/` содержит админ-панель.
- `admin/login.php` выполняет вход в админку.
- `admin/index.php` открывает главную страницу админ-панели.
- `admin/team.php` редактирует существующих сотрудников и позволяет заменить фото сотрудника.
- `admin/services.php` редактирует существующие услуги.
- `admin/prices.php` редактирует существующие цены.
- `admin/logout.php` завершает сессию администратора.
- `admin/api/save-*.php` сохраняют изменения из редакторов.
- `admin/api/upload-team-photo.php` сохраняет новое фото сотрудника и обновляет путь в `data/team.json`.
- `config/admin-auth.php.example` содержит пример локального конфига доступа.
- `config/admin-auth.php` создаётся на хостинге вручную и не коммитится.
- `storage/logs/` используется для audit log.
- `storage/login-attempts.json` используется для rate-limit входа.
- `img/team/` хранит загруженные фото сотрудников и остаётся публичной папкой.
- `includes/*.php` содержит вспомогательный PHP-код публичного рендера.
- `index.html` оставлен как fallback/reference-версия сайта.
- `.htaccess` задаёт приоритет `DirectoryIndex index.php index.html` для Apache-compatible hosting.
- `admin/.htaccess` задаёт приоритет `index.php login.php` внутри `/admin/`.
- `includes/.htaccess` и `storage/.htaccess` закрывают прямой доступ на Apache-compatible hosting.
- `css/`, `js/`, `img/`, `font/` содержат стили, скрипты, изображения и шрифты.

## Что уже готово

- PHP-render через `index.php`.
- JSON-данные в `data/*.json`.
- Админ-панель в `/admin/`.
- Редактор сотрудников.
- Загрузка и замена фото сотрудников через `/admin/team.php`.
- Редактирование Alt и Title для фото сотрудников.
- Редактор услуг.
- Редактор цен.
- Backup JSON перед сохранением.
- Audit log действий администратора.
- CSRF-защита форм сохранения и входа.
- Rate-limit попыток входа.
- Проверка пароля через `password_hash` / `password_verify`.
- Ручной сброс пароля через замену hash в `config/admin-auth.php`.
- Fallback через `index.html`.

## Что не входит в этот пакет

- Загрузка на production.
- Настройка DNS.
- Подключение к конкретной панели хостинга.
- Изменение PHP-кода, JSON, дизайна, SEO или маршрутов.
- Создание новых сущностей в админке.
- Настройка конкретного Nginx `server block`.
- Гарантия защиты служебных файлов без проверки настроек конкретного хостинга.

## Какие файлы сайта нужны на хостинге

В корне сайта должны оказаться:

- `index.php`
- `index.html`
- `.htaccess`
- `admin/`
- `config/`
- `data/`
- `includes/`
- `storage/`
- `css/`
- `js/`
- `img/`
- `font/`

Файл `config/admin-auth.php` создаётся на хостинге из `config/admin-auth.php.example`. В репозиторий его добавлять нельзя.

Загруженные через админку фото сотрудников сохраняются в `img/team/`. Это публичная папка: файлы из неё могут отображаться на сайте. PHP-процессу нужны минимальные права, достаточные для записи в эту папку. Directory listing для `img/team/` запрещён через `img/team/.htaccess`, но на non-Apache hosting этот запрет нужно настроить отдельно.

В репозиторий нельзя добавлять runtime-фото из `img/team/`, реальные backup-файлы `data/backups/**/*.json`, `storage/logs/*.log`, `storage/login-attempts.json` и `config/admin-auth.php`.

## Краткий порядок установки

1. Проверить, что хостинг поддерживает PHP 8.2 или выше, JSON, sessions, загрузку файлов до 3 MB и запись в нужные runtime-файлы.
2. Загрузить файлы сайта в корень сайта на хостинге.
3. Создать `config/admin-auth.php` из `config/admin-auth.php.example`.
4. Сгенерировать `password_hash` для сильного пароля и вставить hash в `config/admin-auth.php`.
5. Убедиться, что `/` открывает PHP-версию через `index.php`.
6. Проверить `/index.php` и `/index.html` напрямую.
7. Проверить, что сотрудники, услуги и цены отображаются из JSON.
8. Проверить вход в `/admin/login.php`.
9. Проверить редакторы `/admin/team.php`, `/admin/services.php`, `/admin/prices.php`.
10. В `/admin/team.php` проверить тестовую замену фото сотрудника.
11. Проверить, что служебные файлы `config/`, `storage/`, `data/backups/` и `includes/` не открываются напрямую, а `img/team/` не показывает directory listing.

Если `/` открывает старый `index.html`, нужно настроить приоритет `index.php` перед `index.html` через хостинг-панель или поддержку хостинга.
