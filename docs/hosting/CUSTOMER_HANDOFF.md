# Передача сайта заказчику

Этот документ - короткая версия для передачи сайта на хостинг.

## Что уже сделано

Сайт подготовлен так, чтобы часть данных можно было брать из JSON-файлов:

- сотрудники берутся из `data/team.json`;
- услуги берутся из `data/services.json`;
- цены берутся из `data/prices.json`.

Главная PHP-версия открывается через `index.php`. Старый `index.html` сохранён как запасной вариант.

## Что нужно от хостинга

Нужен PHP-хостинг:

- PHP 8.2 или выше;
- поддержка JSON в PHP;
- возможность открыть сайт через `index.php`;
- возможность читать JSON-файлы из папки `data/`;
- корректная работа с русским текстом в UTF-8.

Если хостинг не поддерживает PHP, текущая PHP-версия и будущий личный кабинет работать не будут.

## Какие файлы нужно загрузить

На хостинг нужно загрузить:

- `index.php`
- `index.html`
- `.htaccess`
- `css/`
- `js/`
- `img/`
- `font/`
- `data/`
- `includes/`

Файлы должны лежать в корне сайта, то есть там, откуда хостинг открывает главную страницу домена.

## Как проверить сайт после загрузки

Открыть главную страницу сайта и проверить:

- главная страница открывается;
- сотрудники отображаются;
- услуги отображаются;
- цены отображаются;
- карта открывается;
- кнопка оплаты ведёт на прежний адрес;
- русский текст не повреждён;
- на странице нет PHP-ошибок.

Отдельно открыть:

- `/index.php` - PHP-версия сайта;
- `/index.html` - запасная статическая версия.

## Если что-то не открылось

Если открывается `index.html` вместо PHP-версии, нужно попросить хостинг поставить `index.php` выше `index.html` в порядке главных файлов.

Если появляется белый экран или ошибка 500, нужно проверить PHP 8.2+, JSON-расширение и доступ PHP к папке `data/`.

Если вместо сайта показывается PHP-код, значит PHP на хостинге не выполняется.

Если технические файлы из `includes/` открываются напрямую, нужно закрыть прямой доступ к этой папке через настройки хостинга.

## Какие данные сейчас управляются через JSON

Сейчас через JSON управляются:

- сотрудники;
- услуги;
- цены.

Админки пока нет, поэтому редактирование через браузер ещё не реализовано.

## Следующий этап

Следующий этап - личный кабинет. Для него хостингу дополнительно понадобятся авторизация, PHP sessions, безопасная запись в JSON, backup перед сохранением и защита служебных папок.
