# Передача сайта заказчику

Этот документ - короткая версия для передачи сайта и MVP-админки на хостинг.

## Что уже сделано

Сайт подготовлен так, чтобы данные брались из JSON-файлов:

- сотрудники берутся из `data/team.json`;
- услуги берутся из `data/services.json`;
- цены берутся из `data/prices.json`.

Главная PHP-версия открывается через `index.php`. Старый `index.html` сохранён как запасной вариант.

Также готова MVP-админка:

- вход через `/admin/login.php`;
- главная страница админки `/admin/index.php`;
- редактор сотрудников `/admin/team.php`;
- редактор услуг `/admin/services.php`;
- редактор цен `/admin/prices.php`;
- выход через `/admin/logout.php`;
- backup перед сохранением;
- audit log;
- CSRF-защита;
- rate-limit входа.

## Что нужно от хостинга

Нужен PHP-хостинг:

- PHP 8.2 или выше;
- поддержка JSON в PHP;
- PHP sessions;
- возможность открыть сайт через `index.php`;
- возможность читать JSON-файлы из папки `data/`;
- возможность писать в нужные JSON, backup-папки и `storage/`;
- корректная работа с русским текстом в UTF-8;
- возможность закрыть доступ к служебным файлам.

Если хостинг не поддерживает PHP, текущая PHP-версия и админка работать не будут.

## Какие файлы нужно загрузить

На хостинг нужно загрузить:

- `index.php`
- `index.html`
- `.htaccess`
- `admin/`
- `config/`
- `data/`
- `includes/`
- `storage/`
- `css/`
- `js/`
- `img/`
- `font/`

Файлы должны лежать в корне сайта, то есть там, откуда хостинг открывает главную страницу домена.

## Как включить вход в админку

1. Скопировать `config/admin-auth.php.example` в `config/admin-auth.php`.
2. Сгенерировать `password_hash` для нового сильного пароля.
3. Вставить логин и hash в `config/admin-auth.php`.
4. Проверить вход на `/admin/login.php`.
5. Проверить, что `config/admin-auth.php` не открывается из браузера.

Plaintext-пароль нельзя хранить в репозитории, документах или публичных местах.

## Как проверить сайт после загрузки

Открыть главную страницу сайта и проверить:

- главная страница открывается;
- сотрудники отображаются;
- услуги отображаются;
- цены отображаются;
- карта открывается;
- кнопка оплаты ведёт на прежний адрес;
- русский текст не повреждён;
- на странице нет PHP-ошибок.

Отдельно открыть:

- `/index.php` - PHP-версия сайта;
- `/index.html` - запасная статическая версия.

## Как проверить админку

Проверить:

- `/admin/login.php` открывается;
- вход в админку работает;
- `/admin/index.php` открывается после входа;
- `/admin/team.php` позволяет сохранить сотрудников;
- `/admin/services.php` позволяет сохранить услуги;
- `/admin/prices.php` позволяет сохранить цены;
- сохранение создаёт backup;
- сохранение пишет audit log;
- после `/admin/logout.php` закрытые страницы снова требуют вход.

## Что должно быть закрыто от браузера

Нельзя оставлять публичный доступ к:

- `config/admin-auth.php`;
- `storage/`;
- `storage/login-attempts.json`;
- `storage/logs/`;
- `data/backups/`;
- `includes/`.

Если хостинг не Apache, `.htaccess` не сработает. Тогда запрет доступа нужно настроить отдельно через хостинг или поддержку.

## Если что-то не открылось

Если открывается `index.html` вместо PHP-версии, нужно попросить хостинг поставить `index.php` выше `index.html` в порядке главных файлов.

Если появляется белый экран или ошибка 500, нужно проверить PHP 8.2+, JSON-расширение, PHP sessions, права записи и доступ PHP к папке `data/`.

Если вместо сайта показывается PHP-код, значит PHP на хостинге не выполняется.

Если технические файлы из `config/`, `storage/`, `data/backups/` или `includes/` открываются напрямую, нужно закрыть прямой доступ к этим путям через настройки хостинга.

## Какие данные сейчас управляются через админку

Сейчас через админку управляются:

- существующие сотрудники;
- существующие услуги;
- существующие цены.

## Ограничения текущей MVP-админки

Сейчас нельзя:

- создавать или удалять сотрудников;
- загружать фото сотрудников;
- создавать или удалять услуги;
- создавать или удалять цены;
- редактировать rich text / HTML;
- управлять SEO;
- управлять реквизитами;
- управлять картой;
- управлять deploy.

Загрузка фото сотрудников - будущий этап, а не текущая возможность.

## Runtime и backup

Админка создаёт backup перед сохранением:

- `data/backups/team/team-*.json`;
- `data/backups/services/services-*.json`;
- `data/backups/prices/prices-*.json`.

Backup-файлы, audit log, файл попыток входа и `config/admin-auth.php` не коммитятся в репозиторий.

Rollback выполняется вручную на хостинге: перед восстановлением нужно сделать копию текущего JSON-файла, затем заменить его выбранным backup-файлом.
