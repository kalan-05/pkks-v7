# Черновик инструкции по установке сайта

Статус: draft до получения финального домена, данных хостинга и разрешённых доступов.

## Что подготовить перед установкой

- Подтверждённый финальный домен.
- Доступ к панели хостинга или безопасный канал загрузки файлов.
- Информация о корневой папке сайта.
- Подтверждение версии PHP.
- Подтверждение поддержки `.htaccess` и rewrite.
- Подтверждение SSL.
- Решение по SMTP/mail, если отправка почты будет нужна.
- Разрешённые данные для проверки входа в админ-панель.

## Какие файлы загружать

- Публичные файлы сайта: `index.php`, `index.html`, `robots.txt`, `sitemap.xml`, `.htaccess`.
- Папки, необходимые для работы сайта: `admin/`, `config/`, `css/`, `data/`, `font/`, `img/`, `includes/`, `js/`, `storage/`, `vendor/`.
- Только подготовленный clean package, собранный отдельно от рабочей папки разработки.

## Какие файлы не загружать

- `.codex_tmp/**`.
- `debug.log`.
- `.env` и любые файлы с секретами.
- Логи, временные файлы, локальные дампы и резервные копии.
- Служебные отчёты аудита и локальные артефакты разработки.

## Проверка после загрузки

- Открыть `/` и убедиться, что главная страница отдаёт 200.
- Открыть `/index.php` и убедиться, что динамическая версия отдаёт 200.
- Проверить, что CSS, JS, шрифты, изображения и vendor-скрипты загружаются без ошибок.
- Открыть `/admin/login.php` и убедиться, что страница входа доступна.
- Проверить, что защищённые страницы админки без входа перенаправляют на страницу входа или возвращают корректное защитное поведение, не 500.
- Проверить права записи для runtime-папок, которые используются админкой.
- Проверить, что SSL включён и сайт открывается по HTTPS.
- Проверить `/robots.txt` и noindex для `/admin/`.

## OWNER_CONFIRMATION_REQUIRED_FINAL_DOMAIN

Перед финальной установкой заказчик должен подтвердить боевой домен. После подтверждения нужно обновить canonical, sitemap, Host в robots и публичные metadata под реальный домен.

## OWNER_CONFIRMATION_REQUIRED_HOSTING_DETAILS

Перед установкой нужны точные сведения о хостинге: провайдер, панель, PHP version, веб-сервер, `.htaccess`, rewrite, корневая папка, SSL, способ загрузки и права записи.

## OWNER_CONFIRMATION_REQUIRED_ADMIN_CREDENTIALS

Authenticated admin smoke выполняется только после отдельного разрешения владельца и получения временных корректных данных доступа через защищённый канал.
