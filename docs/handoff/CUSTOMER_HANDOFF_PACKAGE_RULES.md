# Правила clean handoff package

## Что входит в clean package

- Публичные HTML/PHP-файлы сайта.
- Рабочие CSS, JS, шрифты, изображения и vendor-файлы, которые реально нужны сайту.
- Админ-панель и серверные include-файлы, необходимые для текущей версии.
- Чистые JSON-данные сайта без runtime-логов и локальных резервных копий.
- Документы передачи из `docs/handoff/`.

## Что не входит в clean package

- Рабочая папка разработки целиком.
- `.codex_tmp/**`.
- `debug.log`.
- `.env` и любые файлы с секретами.
- Runtime logs, backups, temp, локальные дампы и архивы.
- Локальные отчёты аудита, screenshots и технические proof-артефакты.
- Неиспользуемые экспериментальные файлы.

## Правила подготовки

- Готовить отдельный clean archive, а не архивировать текущую рабочую папку.
- Перед упаковкой проверить `git status` и список файлов архива.
- Проверить, что в архиве нет логов, секретов, временных файлов и `.codex_tmp`.
- Проверить, что `/img/og-cover.png` присутствует и не пустой.
- Проверить, что `.htaccess` и `robots.txt` входят в пакет, если хостинг их поддерживает.

## Post-package checklist

- Архив распаковывается в отдельную временную папку.
- В архиве нет `debug.log`, `.env`, `.codex_tmp`, логов, дампов и резервных копий.
- Главная страница открывается локально или на тестовом окружении.
- Админ-панель не индексируется.
- `/debug.log` не отдаётся публично.
- Open Graph image отдаёт 200.
